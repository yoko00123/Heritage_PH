3.2 Main Functions and Buttons


A.  Module Listing

A.1 Header Bar

  1. Module Name
  2. Collapse All / Expand All Button – Used to expand/compress the group details
  3. Add Button – Used to add new record on the module
  4. Delete Button – Used to delete record on the module
  5. Refresh Button – Used re-fetch the list
  6. Column button – Used to manipulate the column header and group header
  7. Print Button
  8. Filter button – Used to filter the record on the list view


A.2 Column Header

  • The description of the data on the row of the list view.

A.3 Group Header

  • Column header which record on the list view are group.


A.4 List view

  • The table where record is listed.
  • Checked box – Used to select multiple record

  • Open button – Used to view the detail of the record on windows from



A.5 Status Bar

  1. Number of record displayed on the list view
  2. Setup for maximum number of displayed record
  3. Page navigation

B. Module Forms

       Forms are used to add, edit and view records.

B. 1 Quick Link Bar

  • Collection of form under record form. Use to navigate on each record form








       B.2 Header Bar





  1. Toggled Button – used to show and hide the Quick Link Bar
  2. Module Name
  3. New Button – Used to open blank record info form
  4. Save Button – Used to save the data on the form
  5. Refresh Button
  6. Print Button
  7. Close Button – Used to close the record info form and display the module listing


B.3 Form Body


    1. Form View – this form is consisted of labels and textbox.



         B. Field Properties

  1. Auto Display / System Generated fields
  • These are fields automatically displayed or generated by the system upon saving or upon providing the necessary value for certain fields.
  • These are fields that cannot be manually edited.


  1. Required fields
  • These are fields needed to be completely filled out by the user.
  • System would prohibit the user from saving any items on the module if the required fields are unfilled.
  • These are distinguished by its Red color.


  1. Optional fields
  • Fields that if left unfilled, the user may still be able to save information on a specific module.
  • These are distinguished by its white/grey color.


  1. Look-up fields
  • These fields contain pop-buttons that contains the list for the subjected field.
  • The user could view the corresponding record by clicking the Open Info button situated within the pop-up form tool bar.
  • These are normally the folder icons.


  1. Date fields
  • These are displayed as textboxes or calendar view that automatically validates and converts the inputted number or string into date (mm-dd-yyyy) format.


  1. Image fields
  • Fields that contains images.
